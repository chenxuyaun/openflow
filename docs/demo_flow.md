# OpenFlow Demo Flow

## Purpose

Provide a complete narration path from first impression to product mechanism so
the same story can be used for landing content, walkthrough demos, and internal
alignment.

## Demo Sequence

### 1. Opening Frame

- Show the landing hero
- State the core promise: every role gets a fresh session, but the project never
  loses context because files carry the state

### 2. The Failure Mode

- Show why long-running chat work breaks down
- Explain that context becomes noise and role transitions lose fidelity

### 3. The OpenFlow Mechanism

- Introduce bootstrap
- Explain workflow graph, task tree, knowledge center, and handoff
- Emphasize that the next role is driven by files, not by hidden conversation state

### 4. First Visible Transformation

- Start with a messy goal
- Show the project dashboard after bootstrap
- Highlight the generated workflow graph and next role

### 5. Role Session Proof

- Open a session detail page
- Show input files, role objective, and structured output
- Show the handoff that recommends the next role

### 6. Knowledge And Auditability

- Open the knowledge center
- Show knowledge items, decisions, and open questions
- Explain how teams can inspect why work moved forward

### 7. Safety And Governance

- Show a confirm gate or review step
- Explain why some transitions are automatic and some require review

### 8. Closing Promise

- Return to the V1 promise
- Restate what the first release lets the user do end to end

## Demo-To-Page Mapping

- Opening frame -> landing hero
- Failure mode -> landing problem
- Mechanism -> landing mechanism and workflow preview
- First visible transformation -> project dashboard and workflow graph
- Role session proof -> session detail
- Knowledge and auditability -> knowledge center
- Safety and governance -> workflow graph and review loop
- Closing promise -> landing V1 promise
