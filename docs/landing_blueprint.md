# OpenFlow Landing Blueprint

## Goal

Turn the product narrative into a first-use entry experience that is clear to
potential users and still consistent with the internal execution model.

## Page Spine

### 1. Hero

- Headline: Every role gets a fresh session. Your project never loses context.
- Subheadline: OpenFlow turns messy goals into visible workflows, reusable
  knowledge, and auditable handoffs.
- Primary promise: Replace hidden chat context with file-based project memory.

### 2. Problem

- Long conversations become noise
- New specialist roles restart from incomplete summaries
- Teams cannot inspect why the next action was chosen

### 3. Mechanism

- Bootstrap turns a goal into roles, tasks, and research artifacts
- Every role works in a new session
- Every session reads explicit files
- Every result becomes the next structured handoff

### 4. Workflow Preview

- User goal
- Bootstrap strategist
- Workflow graph
- Specialist role session
- Handoff and next role
- Review and replanning gate

### 5. Why It Wins

- Better continuity than ordinary chat
- More auditability than opaque agent chains
- More actionability than passive note systems

### 6. V1 Promise

- Bootstrap a project from a single request
- Inspect knowledge, workflow, and decisions
- Open a role session and see what it read
- Review the handoff that drives the next step
